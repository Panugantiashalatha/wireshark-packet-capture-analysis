# Wireshark Packet Capture Analysis

## Project Overview

This project demonstrates basic network traffic analysis using **Wireshark**, a widely used network protocol analyzer. The purpose of this task was to capture live network packets, apply protocol filters, and analyze the communication occurring between the local system and external servers.

The packet capture included normal network activities such as web browsing, local network discovery, time synchronization, and encrypted communication.

---

# Objectives

* Capture live network traffic using Wireshark
* Identify different network protocols
* Analyze packet flow and communication behavior
* Understand protocol usage in real-world network traffic

---

# Tools Used

* **Wireshark**
* Operating System: Linux / Virtual Machine Environment
* Network Interface: `eth0`

---

# Task Steps

## 1. Installation

Wireshark was installed successfully on the system.

## 2. Packet Capture

Packet capture was started on the active network interface:

```bash
eth0
```

## 3. Traffic Generation

Network traffic was generated by:

* Browsing websites
* Connecting to external servers
* Allowing system background services to communicate

## 4. Capture Duration

Traffic was captured for approximately **1 minute**.

## 5. Protocol Filtering

Filters were applied to analyze specific protocols:

```plaintext
tcp
arp
ntp
icmp
tls
```

## 6. Protocol Identification

Multiple protocols were identified during packet analysis.

---

# Captured Protocols

## 1. TCP (Transmission Control Protocol)

TCP traffic was observed between the local machine and multiple remote servers.

### Example

* Local IP: `192.168.235.141`
* Remote IPs:

  * `34.107.243.93`
  * `65.61.137.117`

### Observed Activity

* SYN packets (connection initiation)
* ACK packets (acknowledgment)
* FIN packets (connection termination)
* RST packets (connection reset)

TCP traffic mainly included:

* HTTP communication (Port 80)
* HTTPS communication (Port 443)

---

## 2. ARP (Address Resolution Protocol)

ARP traffic was observed for local network communication.

### Example

```plaintext
Who has 192.168.235.27? Tell 192.168.235.141
```

### Purpose

ARP resolves:

* IP addresses → MAC addresses

### Observed Activity

* Broadcast requests
* Device replies

This indicates normal local network device discovery.

---

## 3. NTP (Network Time Protocol)

NTP traffic was detected between the local system and an external time server.

### Example

* Local IP: `192.168.235.141`
* NTP Server: `95.216.144.226`

### Purpose

NTP is used for:

* Time synchronization
* Maintaining accurate system time

---

## Additional Protocols Observed

### ICMP (Internet Control Message Protocol)

ICMP packets were observed in the form of router solicitation messages.

### Purpose

* Router discovery
* Network diagnostics

---

### TLS v1.2 (Transport Layer Security)

Encrypted communication with external servers was observed.

### Purpose

* Secure HTTPS communication
* Protect data during transmission
